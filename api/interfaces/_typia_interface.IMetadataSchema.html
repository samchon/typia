    Interface IMetadataSchema

    Metadata schema representing a TypeScript type's structure.

    IMetadataSchema is typia's internal type representation, capturing full TypeScript type information including unions, optionality, nullability, and type constraints. Used by typia.reflect.schema<T>() for runtime type introspection.

    Type categories:

    • Primitives: atomics (boolean, bigint, number, string)
    • Literals: constants (literal values like "hello" or 42)
    • Templates: templates (template literal types)
    • Collections: arrays, tuples, sets, maps
    • Objects: objects (named object types)
    • Aliases: aliases (type aliases)
    • Natives: natives (built-in classes like Date, Uint8Array)
    • Functions: functions (function types)
