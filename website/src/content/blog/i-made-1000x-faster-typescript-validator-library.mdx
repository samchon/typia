Hello, I'm developer of [typescript-json](https://github.com/samchon/typescript-json) and have measure a benchmark comparing performance with other competitive libraries. By the benchmark, my library [typescript-json](https://github.com/samchon/typescript-json) has been turned out that maximum 1,000x times faster than [zod](https://github.com/colinhacks/zod).

Being proud of such performance enhancement, I hope many TypeScript developers to adapt my library. Advantage of [typescript-json](https://github.com/samchon/typescript-json) is not only better performance, but also easy usage exists. [typescript-json](https://github.com/samchon/typescript-json) does not require any extra schema definition and just requires only one line with pure TypeScript Type. Thus, what about using my library [typescript-json](https://github.com/samchon/typescript-json) instead of [io-ts](https://github.com/gcanti/io-ts) or [zod](https://github.com/colinhacks/zod)?

```typescript
// ALLOW SUPERFLUOUS PROPERTIES
export function is<T>(input: T): boolean; // true or false
export function assertType<T>(input: T): T; // throws `TypeGuardError`
export function validate<T>(input: T): IValidation; // detailed reasons

// DO NOT ALLOW SUPERFLUOUS PROPERTIES
export function equals<T>(input: T): boolean;
export function assertEquals<T>(input: T): T;
export function validateEquals<T>(input: T): IValidation;
```

## `is()` function
```typescript
export function is<T>(input: T): boolean;
```

![Is Function Benchmark](https://user-images.githubusercontent.com/13158709/194713658-62fb0716-c24e-41f9-be0d-01edeabb1dd6.png)

> Measured on Intel i5-1135g7, Surface Pro 8

I hadn't known that [zod](https://github.com/colinhacks/zod) is extremely slow until running on this benchmark program. [io-ts](https://github.com/gcanti/io-ts) is also slower than [typescript-json](https://github.com/samchon/typescript-json) and [ajv](https://ajv.js.org) about 8x times, but its slow performance looks like nothing comparing with [zod](https://github.com/colinhacks/zod).

More amazing thing is, [ajv](https://ajv.js.org) requires JSON schema definition for validating, but it can't validate the JSON schema type itself. Although other libraries [io-ts](https://github.com/gcanti/io-ts) and [zod](https://github.com/colinhacks/zod) also failed to validating the JSON schema definition, isn't [ajv](https://ajv.js.org) different with them? Furthermore, when complicate type comes, [ajv](https://ajv.js.org) dies with runtime error in the most case.



## `validate()` function
```typescript
export function validate<T>(input): IValidation;

export interface IValidation {
    success: boolean;
    errors: IValidation.IError[];
}
export namespace IValidation {
    export interface IError {
        path: string;
        expected: string;
        value: any;
    }
}
```

![Super-fast runtime validator](https://user-images.githubusercontent.com/13158709/194713627-59c12e51-06e8-44ba-9df5-cfd69aaf971a.png)

> Measured on Intel i5-1135g7, Surface Pro 8

Unfortunately, unable to measure [ajv](https://ajv.js.org)'s performance because it does not support the validate function describing detailed error reasons.

Anyway, although [zod](https://github.com/colinhacks/zod) is the slowest library even in the `validate()` function, performance gap with other libraries are dramatically(?) reduced. Also, performance gap ratio with [typescript-json](https://github.com/samchon/typescript-json) with [io-ts](https://github.com/gcanti/io-ts) are sometimes increased and sometimes reduced.

By the way, when measuring of [zod](https://github.com/colinhacks/zod), I sometimes doubted myself, "Have I taken a mistake?". But there was not any mistake and I wondered why [zod](https://github.com/colinhacks/zod) is such slow. To analyze the reason why, I'd read source code of [zod](https://github.com/colinhacks/zod) and this is my conclusion.

> Although [zod](https://github.com/colinhacks/zod) is the slowest validator, error reasons are more detailed and exact than [io-ts](https://github.com/gcanti/io-ts). Therefore, its slow performance on `validate()` function is understandable.
>
> Also, [zod](https://github.com/colinhacks/zod) hasn't implemented special logic for `is()` function and let user to re-use the `validate()` function. Therefore, when [zod](https://github.com/colinhacks/zod) implements the independent `is()` function algorithm, I think its performance can overcome [io-ts](https://github.com/gcanti/io-ts).
